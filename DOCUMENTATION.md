## Detailed Skills List

### 🔧 Development Skills (10)

#### 1. spec-driven-development-enhanced
**4-phase gated workflow with assumption surfacing and drift detection**

- Phase 1: Spec Creation (requirements, assumptions, acceptance criteria)
- Phase 2: Implementation Planning (architecture, dependencies, risks)
- Phase 3: Development (incremental, test-driven, verified)
- Phase 4: Validation (acceptance tests, spec compliance, sign-off)

**When to use:** New features, complex changes, team projects

#### 2. tdd-iron-law
**Strict TDD enforcement with AST-based test mapping and mutation testing**

- RED: Write failing test first
- GREEN: Make it pass (minimal code)
- REFACTOR: Clean up while keeping tests green
- VERIFY: Mutation testing to validate test quality

**When to use:** Critical code, bug fixes, refactoring

#### 3. security-review-owasp
**OWASP Top 10 2025 + Agentic AI Top 10 2026 security review**

- Static analysis (code patterns, secrets detection)
- Dynamic analysis (runtime behavior, API testing)
- Architecture review (design patterns, trust boundaries)
- Report & remediate (findings, fixes, verification)

**When to use:** Before commits, PRs, production deploys

#### 4. architecture-blueprint
**C4 diagrams, ADRs, technology radar, deployment topology**

- Context diagram (system boundaries)
- Container diagram (high-level tech)
- Component diagram (internal structure)
- Code diagram (class/module level)
- ADRs (architectural decisions)

**When to use:** New projects, major refactors, documentation

#### 5. parallel-orchestration
**Multi-agent coordination and task decomposition**

- Task decomposition (break into parallel work)
- Agent assignment (match skills to tasks)
- Coordination (handoffs, dependencies)
- Integration (merge results, verify)

**When to use:** Large features, research tasks, parallel work

#### 6. incremental-implementation
**Tiny steps with continuous verification**

- Break into smallest possible steps
- Implement one step at a time
- Verify after each step
- Never move forward with broken code

**When to use:** Complex features, unfamiliar code, debugging

#### 7. context-engineering
**10x token efficiency optimization**

- Load exactly what you need
- Use search before reading full files
- Paginate large files
- Cache frequently accessed content
- Prune stale context

**When to use:** Large codebases, token limits, slow responses

#### 8. git-workflow-advanced
**Git Flow, GitHub Flow, trunk-based development**

- Branching strategies
- Commit conventions
- PR workflows
- Release management
- Conflict resolution

**When to use:** Team projects, open source, production code

#### 9. database-optimization
**7-layer DB optimization (indexing, queries, scaling)**

- Layer 1: Query optimization
- Layer 2: Indexing strategy
- Layer 3: Schema design
- Layer 4: Connection pooling
- Layer 5: Caching
- Layer 6: Replication
- Layer 7: Sharding

**When to use:** Slow queries, high load, scaling issues

#### 10. code-quality-metrics
**8-metric quality framework (complexity, coverage, debt)**

- Cyclomatic complexity
- Test coverage

- Code smells

**When to use:** Code reviews, refactoring, quality tracking

## Integration Guide

### For Hermes Agent

```bash
# Copy skills
cp -r udon-collective-skills/skills/* ~/.hermes/skills/

# Load a skill
hermes skill view spec-driven-development-enhanced
```

### For OpenCode

```bash
# Copy skills
```
